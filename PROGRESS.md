## Notes / decisions

- **Bulk out-of-scope handling (criteria 16 vs 20 — spec is slightly
  self-contradictory here).** Hard Rule: RLS is never disabled. So a row whose
  facility/category is outside a *scoped* role's DB permissions genuinely cannot
  be inserted by that role. Resolution implemented: (a) rows with data-validity
  issues (bad date, blank quality, unit mismatch, ambiguous source) DO insert
  with `requires_review = true` — the Analyst is the primary bulk uploader and
  for them everything is in scope, so these all land flagged in the Review Queue
  (criterion 16 path); (b) rows a scoped role has no DB permission for are shown
  in the preview as "Outside your scope" and are **not silently dropped** — they
  are surfaced, excluded from the batch, and if one slips through it is rejected
  by RLS and reported per-row (criterion 20). This keeps both "never disable RLS"
  and "never silently drop" true. See D-2's Limitation.
- **Natural gas in litres** (present in the sample) is flagged `unit_mismatch`:
  canonical unit is m³ and there is no safe fixed litres→m³ factor for a gas, so
  the value is stored unconverted with the raw unit and routed to review rather
  than guessed. LPG litres→kg (0.51) is the one real conversion in the sample.
- Auto-flag rule (spec Section 5 "automatic rule"): non-positive activity values
  are auto-flagged `requires_review = true` on insert (`auto_flag_activity`).
  Adjust in `0001_schema.sql` if a different rule is preferred.
- `dim_emission_factor` / `dim_methodology` seeded as PLACEHOLDERS — G-4 replaces.
- Lookup PKs are readable text codes to keep seed data + joins legible.
